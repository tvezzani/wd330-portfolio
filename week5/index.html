Errors, exceptions, warnings

  • Error
    Something goes wrong with the system, the programmer(syntax or logic) or the user input.
  • Exceptions
    Something goes wrong with the system that you do not have control over and need to be prepared for. 3rd party services are common causes of exception errors due to the nature of depending on outside code.
  • Warnings
    Messages will be provided to the user to notify them of soft errors that wont crash the code. An example is when a certain functionality is deprecated, and will need to be replaced in the future.

The importance of testing and debugging

  • Debugging is essential to fixing inevitable problems that will show up when implementing code.
  • Testing helps developers to catch and avoid most bugs in code

Using the Console

  • console.log()
  • console.warn() for depreciated features
  • console.error() for errors (invalid credentials)

Debugging Tools

  • debugger keyword creates a breakpoint in your code
  • breakpoints help streamline error checking

Error objects

  • error objects are error specific objects and there are different ones to choose from.
  • The eval() function evaluates code
  • Each error has a message property

Throwing exceptions

  • using the throw keyword you can report an error in the console
  • it is best practice to throw error objects
  • catching an error prevents it from bubbling up in your console

Exception handling

  • try and catch are used in a function that catches an error
  • A finally can be added after a catch block. It is run no matter what.

Tests

  • It is helpful to use tests to help guide code development

Test-driven Development

  • The testing framework is created before any actual code
  • One the testing is streamlined, the code is written until the tests pass

Testing frameworks

  • There are several testing frameworks, including Jest
  • using the test() function, you can create a test, which is used to show a pass or fail
